Detailed Description

template<class Scalar>
class EigenOpt::quadratic_programming::Solver< Scalar >

Quadratic Programming solver using active set and null-space projections.

This class implements a quadratic programming solver to minimize a given cost function as in:

\begin{equation} \min_{\mathrm{\mathbf{x}}} \left\| \mathrm{\mathbf{Q}} \mathrm{\mathbf{x}} - \mathrm{\mathbf{r}} \right\|^2 \quad\text{subject to:}\quad \begin{cases} \mathrm{\mathbf{A}} \mathrm{\mathbf{x}} = \mathrm{\mathbf{b}} \\ \mathrm{\mathbf{C}} \mathrm{\mathbf{x}} \leq \mathrm{\mathbf{d}} \end{cases} \end{equation}

The reason for using such a specific form for the objective is that it is very well suited for robotics applications, where mathematical models often are written as \( \dot{\mathrm{\mathbf{s}}} = \mathrm{\mathbf{J}} \dot{\mathrm{\mathbf{q}}} \), where \( \mathrm{\mathbf{s}} \) is a vector of features while \( \mathrm{\mathbf{q}} \) are the generalized coordinates of the system - whose derivatives often serve as inputs for the system. Advanced control laws can thus be formulated using quadratic programming.

To determine a solution for the problem, the first step is to remove the equalities via a null-space projection strategy. The system of equalities is solved to find a particular solution \(\mathrm{\mathbf{x}}_{eq}\) such that \( \mathrm{\mathbf{A}} \mathrm{\mathbf{x}}_{eq} = \mathrm{\mathbf{b}} \). Then, an orthonormal basis of the kernel of \( \mathrm{\mathbf{A}} \) - denoted as \( \mathrm{\mathbf{Z}} \) - is computed. By definition, its property is that \( \mathrm{\mathbf{A}}\mathrm{\mathbf{Z}} = 0 \). The original decision vector is thus substituted in the original problem using the parameterization

\begin{equation} \mathrm{\mathbf{x}} = \mathrm{\mathbf{x}}_{eq} + \mathrm{\mathbf{Z}} \mathrm{\mathbf{y}} \end{equation}

where \( \mathrm{\mathbf{y}} \) is a new, lower-dimensional, decision vector. Note that no matter its value, equality constraints will be satisfied. The problem now becomes:

\begin{equation} \min_{\mathrm{\mathbf{y}}} \left\| \mathrm{\mathbf{Q}}_y \mathrm{\mathbf{y}} - \mathrm{\mathbf{r}}_y \right\|^2 \quad\text{subject to:}\quad \mathrm{\mathbf{C}}_y \mathrm{\mathbf{y}} \leq \mathrm{\mathbf{d}}_y \end{equation}

where:

\begin{align} \mathrm{\mathbf{Q}}_y &\doteq \mathrm{\mathbf{Q}}\mathrm{\mathbf{Z}} \\ \mathrm{\mathbf{r}}_y &\doteq \mathrm{\mathbf{r}} - \mathrm{\mathbf{Q}}\mathrm{\mathbf{x}}_{eq} \\ \mathrm{\mathbf{C}}_y &\doteq \mathrm{\mathbf{C}}\mathrm{\mathbf{Z}} \\ \mathrm{\mathbf{d}}_y &\doteq \mathrm{\mathbf{d}} - \mathrm{\mathbf{C}}\mathrm{\mathbf{x}}_{eq} \end{align}

The problem can be solved using an active-set strategy, whose main steps are:

  1. Determine an initial feasible solution that satisfies all inequalities.
  2. Parameterize the decision vector as \( \mathrm{\mathbf{y}} = \mathrm{\mathbf{y}}_k + \mathrm{\mathbf{p}} \), where \( \mathrm{\mathbf{y}}_k \) is the current solution and \( \mathrm{\mathbf{p}} \) is a "step".
  3. Given a set of "active constraints", compute \( \mathrm{\mathbf{p}} \) so that it minimizes the objective while enforcing the active constraints.
  4. Rather than jumping to the new iterate immediately, now consider the line parameterization \( \mathrm{\mathbf{y}} = \mathrm{\mathbf{y}}_k + \alpha \mathrm{\mathbf{p}} \) where \( 0 \leq \alpha \leq 1 \). The value of this coefficient is evaluated as the largest possible value that does not cause any new constraint to be violated. If the final value is less than 1, it means that a new constraint has just activated andi t is thus added to the active set. If \( \alpha = 1 \), then the full step can be performed without adding new constraints to the active set.
  5. At the new point, it is possible to compute the Lagrange multipliers associated to the active constraints. If any multiplier is negative, it means that the constraint does not need to be enforced, and it can be removed from the active set.
  6. Given the new solution and active set, a new iteration is performed from step 2. The algorithm stops when \( \alpha = 1 \) and no constraints deactivate.

Update inequality constraints to the problem.

Existing equality constraints will not be removed. If the constraint dimensions have not changed, the active set will not be reset and feasibility is not tested either. The reason for this method to exists is to help solving multiple similar (but not identical) problems sequentially, warm-starting each one with the information obtained in the previous problem.

Parameters
CInequalities constraints matrix.
dInequalities constraints vector.
Returns
true if the constraints did not change dimension, or if they did and the new ones are feasible.
